.env FILE RECOVERY — cPanel (api.aljaroshi.com)
==============================================

Your .env was deleted. Create a NEW file named exactly:  .env
Location: same folder as package.json and dist/ (api subdomain root)

STEP 1 — Get MySQL details from cPanel
---------------------------------------
cPanel > MySQL Databases
- Database name (e.g. aljarosh_crm)
- Database user (e.g. aljarosh_user)
- Password (the one you set for that user)
- Host is usually: localhost

STEP 2 — Create .env in File Manager
------------------------------------
cPanel > File Manager > api.aljaroshi.com folder
> + File > name it: .env
> Edit > paste the template below (replace YOUR_* values)

STEP 3 — Paste this (edit YOUR values):
---------------------------------------

DATABASE_URL="mysql://YOUR_DB_USER:YOUR_DB_PASSWORD@localhost:3306/YOUR_DB_NAME"
JWT_SECRET="AlJaroshi-ServiceOS-2026-ChangeThisToRandom32Chars"
PORT=5000
FRONTEND_URL="https://crm.aljaroshi.com"
NODE_ENV=production

EXAMPLE (fake values — use YOUR real ones):
DATABASE_URL="mysql://aljarosh_crmuser:MyP@ssw0rd@localhost:3306/aljarosh_serviceos"

IMPORTANT:
- If password has special characters (@ # % etc), URL-encode them in DATABASE_URL
  @ becomes %40   # becomes %23   % becomes %25
- JWT_SECRET: if you use a NEW secret, old login tokens stop working (users re-login — OK)
- If you remember the OLD JWT_SECRET, use the same one so nothing breaks

STEP 4 — After saving .env
---------------------------
SSH or cPanel Terminal:
  cd /home/YOURUSER/api.aljaroshi.com
  npm install --production
  npx prisma generate

cPanel > Setup Node.js App > RESTART

STEP 5 — Test
-------------
Browser: https://api.aljaroshi.com/health
Must show: {"status":"ok","message":"Server is healthy"}

Then login at: https://crm.aljaroshi.com/login

IF YOU FORGOT DB PASSWORD:
cPanel > MySQL Databases > Change password for the database user
Then update DATABASE_URL in .env

IF YOU FORGOT JWT_SECRET:
Use any new long random string — only effect is everyone must login again.
